📊 各层详细说明

Layer 1: 数据基础层（Unified Telemetry）

提供统一的、高质量的、可关联的遥测数据，是AI诊断的根基。
核心能力：
• 指标数据（Metrics）：QPS、延迟、错误率、饱和度
• 日志数据（Logs）：应用日志、系统日志、错误堆栈
• 链路数据（Traces）：分布式调用链、Span级别的性能数据
• 变更数据（Changes）：代码发布、配置变更、基础设施变更
• 拓扑数据（Topology）：服务调用关系、依赖图谱

Layer 2: 智能发现层（Intelligent Detection）

告警进入诊断流程前的预处理层，解决告警风暴、重复告警、低价值告警等问题。
核心功能：告警去重、告警关联、动态基线、告警分级、上下文富化

Layer 3: 根因诊断层（Root Cause Analysis）

多Agent并行诊断，定位根因。
五大诊断Agent：
• Metrics Agent：指标分析、异常检测、相关性分析
• Logs Agent：日志分析、错误聚类、异常检测
• Change Agent：变更关联、变更分析、回滚建议
• Topology Agent：拓扑发现、影响分析、根因溯源
• Knowledge Agent：知识检索、模式匹配、经验推荐

Layer 4: 智能处置层（Intelligent Remediation）

根据诊断结果自动或半自动执行修复动作，缩短MTTR。
处置策略分级：
• 低风险：自动执行（重启、扩容、清理日志）
• 中风险：建议+一键执行（配置回滚、流量切换）
• 高风险：人工审批（数据库切换、架构变更）

Layer 5: 复盘与学习层（Postmortem & Learning）

自动复盘、知识沉淀、持续优化。
核心能力：
• 诊断准确性评估、建议采纳率追踪、NPS评分收集
• 故障模式库、解决方案库、诊断案例集沉淀
• 持续学习闭环：Bad Case分析 → 模型重训练 → A/B测试 → 全量发布
